# Agent Notes

## Commands
- Build the bot with `make build`; it compiles `./cmd/inanny/main.go` to `./build/app`.
- Run locally with `make run` or `go run ./cmd/inanny/main.go`.
- Agents do not need to write tests unless the user explicitly asks for them.

## Runtime Configuration
- The bot requires `TELEGRAM_BOT_TOKEN`; startup exits if it is missing.
- Database access reads `DB_HOST`, `DB_USER`, `DB_PASSWORD`, and `DB_NAME` directly from the environment.
- `local/` is ignored by git and is the safest place for untracked local env or scratch files.

## Database And Generated Code
- SQL schema lives in `migrate/tables/`; SQL queries live in `migrate/queries/`.
- `internal/generated/queries/` is generated by `sqlc`; do not edit it by hand.
- After changing migration tables or query SQL, regenerate with `sqlc generate -f config/sqlc.yaml`.
- Apply migrations with `make db-migrate`; it requires Liquibase plus `libs/postgresql-42.7.5.jar` and the DB env vars above.

## Project Structure
- Real app entrypoint: `cmd/inanny/main.go`, which calls `internal/inanny/bot.StartBot()`.
- Telegram commands are registered in `internal/inanny/bot/handlers/commands/commands.go`; adding a command handler also requires adding it to `commandHandlers`.
- Docker builds the same entrypoint and runs `/app/main`; `make docker-run` passes only the Telegram and DB env vars listed above.
